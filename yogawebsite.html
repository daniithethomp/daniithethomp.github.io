Sarah's Go With The Flow Yoga

Visit Site

A full stack website that allows yoga classes and courses to be posted by an admin user and students to book and pay for these classes.

Features

  • Admin Management
  • Class creation
  • Online payments through Stripe
  • Email confirmation of bookings
  • Booking management
  • Email contact form

Technologies

  • Python
  • HTML
  • CSS
  • Flask
  • Jinja
  • SMTP
  • Stripe API
  • Render web hosting

The website was designed to be simplistic and minimalistic so that the user feels calm and not overwhelmed when browsing the page. The website was designed with desktop use in mind, but mobile design has also been considered with breakpoints in the CSS when the screen size is small. Every page on the website is accessible within 3 clicks to make sure that it is easy to navigate.

The home page displays information about the Yoga Instructor and their studio at the request of the business owner. The animation displayed upon loading the home page was created purely using CSS.

The classes page displays a list of all future classes that the instructor has posted and displays basic information about each class. The user can then click on a class to view more information and to book the class.

Displays the full information of the class (date, time, duration, price) and the instructor set description. It then allows a user to enter their email address to be taken to the payment portal. The price of the class is retrieved by making an API call using the price ID of the Stripe product.

Payments are taken through Stripe which upon completion uses a webhook to fulfill the booking (add the booking to the database and email the user information about their order).

When adding bookings to the database a transaction is started to ensure database integrity.

The email has both HTML and plain text versions depending on what the email supports, and is sent via SMTP using python's SMTPlib and email libraries.

The contact form sends an email to the business owner with the user's input name, email and message so that they can contact the user. This again uses pythons SMTP and email libraries to send the email over SMTP.

Admins can create classes by manually inputting the title, description, time, date, length, number of available spaces and the Stripe product price ID. These classes can then be edited to change any details that need to be changed.

On the same page that you edit a class, admins can also view the emails of those who have booked the class and delete or manually create bookings as they need.

To host the website I used Render as it allows for automatic deployment using Github, custom environment variables, ssh and shell connection, persistent disks and one off jobs. It is also very simple to use and is well documented.
